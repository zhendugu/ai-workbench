## 0. 设计目标（不可更改）

本 Run-Time 前端的唯一目的，是帮助人类**阅读、检查、审计**一个已冻结的 AI 虚拟公司结构。

**本前端的本质**：
- 是结构阅读工具
- 是审计界面
- 是冻结快照的查看器
- 不是执行系统
- 不是监控工具
- 不是控制台
- 不是仪表盘
- 不与 AI API 交互
- 不显示任何运行时数据

**核心设计前提**：
Run-Time 不是"系统正在运行"。
Run-Time 是"人类正在阅读一个冻结的结构"。

**界面感觉应该接近**：
- 阅读冻结的法律实体档案（或历史性法律实体记录）
- 检查冻结的建筑蓝图
- 审计声明的结构
- 查看历史文档

**界面感觉不应该接近**：
- 仪表盘
- 控制面板
- 监控工具
- 工作流引擎
- 实时数据可视化

**成功标准**：
人类在使用后应能清晰回答：
- "这个冻结的公司结构是什么？"
- "每个 Cell 的责任边界在哪里？"
- "Cell 之间的声明性关系是什么？"
- "这个结构是什么时候被冻结的？"
- "谁冻结了这个结构？"
- "这个结构为什么被冻结？"

人类不应产生的感觉：
- "系统正在运行"
- "数据正在更新"
- "有活动正在进行"
- "系统在帮我做决定"
- "系统在推荐我做什么"

## 1. Run-Time 与 Design-Time 的严格区分

### 1.1 Run-Time 状态特征

**允许的操作**：
- 查看冻结后的 Company 结构
- 阅读 Cell 责任描述和边界
- 查看 Role 约束
- 浏览声明性拓扑关系
- 查看方法论视角
- 回溯设计决策
- 引用 Company 标识符
- 导出结构快照（如果已实现）

**禁止的操作**：
- 任何编辑
- 任何修改
- 任何删除
- 任何执行
- 任何启动或停止
- 任何刷新（暗示活动）
- 任何状态切换

**状态标识**：
- 界面顶部必须持续显示：【RUN-TIME - READ ONLY】
- Company 显示状态：Frozen
- 所有字段为只读文本
- 显示冻结时间戳
- 显示冻结人标识
- 显示冻结快照 ID

**明确说明的事实**：
在界面顶部必须显示：
> "This structure is frozen and cannot be modified. This is a read-only view of a structural declaration."

---

### 1.2 Design-Time 与 Run-Time 的切换

**切换规则**：
- Run-Time 界面**不能**直接跳转到 Design-Time 编辑
- 如需修改，必须创建新的 Draft（在 Design-Time 界面中）
- Run-Time 界面可以显示"基于此 Frozen Company 创建新 Draft"的引用链接（只读，不触发操作）

**明确要求**：
- Design-Time 和 Run-Time 必须是完全独立的界面模式
- 切换必须通过明确的导航（如返回 Design-Time 主界面）
- 不能在同一界面中混合 Draft 和 Frozen 状态

## 2. Run-Time 信息层级定义

### 2.1 信息层级结构

**重要说明**：以下信息层级是**阅读视角的信息组织方式**，不是数据层级、监控层级或执行层级。它们仅用于帮助人类理解冻结结构的组织方式，不暗示任何活动、状态变化或实时数据。

```
L1: Company Identity Layer（公司身份层）
    ├─ Company 标识符（Frozen ID）
    ├─ Company 名称和描述
    ├─ 冻结时间戳
    ├─ 冻结人标识
    └─ 冻结快照 ID

L2: Structure Layer（结构层）
    ├─ Cell 列表（只读）
    ├─ Cell 详情（只读）
    │   ├─ 责任描述（What）
    │   ├─ 责任边界（What NOT）
    │   └─ 附属 Role 约束（只读）
    └─ 结构完整性事实（如孤立 Cell 数量）

L3: Topology Layer（拓扑层）
    ├─ 声明性拓扑画布（只读）
    ├─ 关系列表（只读）
    └─ 关系类型和描述（只读）

L4: Methodology Context Layer（方法论上下文层）
    ├─ 冻结时采用的方法论
    ├─ 方法论描述
    └─ 方法论状态（冻结时）

L5: Provenance / Freeze Record Layer（来源/冻结记录层）
    ├─ 冻结时间戳
    ├─ 冻结人标识
    ├─ 冻结原因（如果有记录）
    ├─ 冻结前 Draft ID（如果有）
    └─ 版本链引用（如果有）
```

### 2.2 层级可见性规则

**始终可见**：
- L1: Company Identity Layer（必须在所有视图中可见）

**上下文可见**：
- L2: Structure Layer（在查看结构时可见）
- L3: Topology Layer（在查看拓扑时可见）
- L4: Methodology Context Layer（在查看方法论时可见）
- L5: Provenance Layer（在查看冻结记录时可见）

**禁止**：
- 隐藏关键身份信息
- 隐藏冻结时间戳
- 隐藏"READ ONLY"标识

## 3. Run-Time 实体呈现规范

### 3.1 Company（冻结公司）

**实体定义**：
Company 是冻结结构的语义锚点，用于回答"我在查看哪个冻结的虚拟公司"。

**必须呈现的字段**（全部只读）：
- Company 标识符（Frozen ID，格式如 `FROZEN-COMP-{UUID}`）
- Company 名称（只读文本）
- Company 描述（只读文本）
- 冻结状态标签：**Frozen**（只读，不可编辑）
- 冻结时间戳（只读）
- 冻结人标识（只读）
- 冻结快照 ID（只读，用于引用）

**呈现位置**：
- 界面顶部或左侧固定区域
- 必须在所有 Run-Time 视图中持续可见

**明确标识**：
- 必须显示 "FROZEN COMPANY - READ ONLY" 文字标签
- 必须显示冻结时间戳
- 必须显示"此结构无法修改"的明确说明

**禁止元素**：
- 编辑按钮
- 修改链接
- 状态切换控件
- 刷新按钮（暗示活动）
- 任何暗示"系统正在运行"的元素

**明确说明的事实**：
在 Company 信息区域顶部必须显示：
> "This is a frozen structural declaration. It cannot be modified. To make changes, create a new Draft in Design-Time."

---

### 3.2 Cell（责任单元 - 只读）

**实体定义**：
Cell 是冻结的责任声明单元。在 Run-Time 中，Cell 仅用于展示，不用于任何操作。

**必须呈现的字段**（全部只读）：
- Cell 标识符（格式如 `CELL-{UUID}`）
- Cell 名称（只读文本）
- 责任描述（What）（只读文本）
- 责任边界（What NOT）（只读文本）
- 附属 Role 约束列表（只读，见 3.3）

**呈现方式**：

**方式 A：列表视图**
- 以卡片或表格形式呈现所有 Cell
- 每个卡片显示：名称 / 责任摘要 / 边界摘要
- 点击进入详情查看（只读）

**方式 B：网络图视图**
- 以节点形式呈现 Cell（只读）
- 节点内显示 Cell 名称
- 节点之间的连线表示声明性关系（只读）
- 点击节点进入详情查看（只读）

**操作允许**：
- 查看 Cell 详情（只读）
- 展开/折叠 Role 约束列表（仅显示控制）
- 导航到相关关系（只读）

**禁止操作**：
- 编辑任何字段
- 删除 Cell
- 修改 Role
- 任何暗示"Cell 正在工作"的显示

**去拟人化要求**：
- 禁止使用头像图标
- 禁止使用"成员" "员工" "Agent"等词汇
- 禁止显示"执行状态"或"活跃状态"
- 界面文案使用"责任单元"而非"角色"或"成员"

**明确说明的事实**：
在 Cell 列表或详情页顶部必须显示：
> "Cell is a frozen responsibility declaration. It is not an executor and does not run."

---

### 3.3 Role（约束 - 只读）

**实体定义**：
Role 是 Cell 的附属约束描述（只读）。在 Run-Time 中，Role 仅用于展示约束条件，不用于任何操作。

**必须呈现的字段**（全部只读）：
- Role 名称（只读文本）
- 约束类型（只读：允许 / 禁止 / 条件）
- 约束描述（只读文本）

**呈现位置**：
- 必须在 Cell 详情页内呈现（只读）
- 以列表或折叠区域形式附属于 Cell
- 不可作为一级导航项

**操作允许**：
- 查看 Role 详情（只读）
- 展开/折叠 Role 列表（仅显示控制）

**禁止操作**：
- 编辑 Role
- 删除 Role
- 移动 Role
- 任何暗示"Role 正在生效"的显示

**明确说明的事实**：
在 Role 区域顶部必须显示：
> "Role is a frozen constraint condition attached to this Cell. It is not an independent entity."

---

### 3.4 Topology（声明性拓扑 - 只读）

**实体定义**：
Topology 是 Cell 之间的冻结声明性关系结构。在 Run-Time 中，Topology 仅用于展示结构关系，不用于任何操作。

**必须呈现的关系类型**（全部只读）：
1. **依赖关系**：Cell A 依赖 Cell B 的输出
2. **参考关系**：Cell A 参考 Cell B 的定义
3. **输入输出关系**：Cell A 的输出是 Cell B 的输入

**呈现方式**：

**方式 A：图形拓扑画布（只读）**
- 节点：Cell（显示名称，只读）
- 边：关系类型（带文字标签，只读）
- 布局：冻结时的布局（不可拖拽）
- 交互：
  - 点击 Cell 进入详情查看（只读）
  - 点击边查看关系详情（只读）
  - **禁止拖拽**（暗示可编辑）

**方式 B：关系列表（只读）**
- 表格形式显示所有关系（只读）
- 列：源 Cell / 关系类型 / 目标 Cell / 描述

**操作允许**：
- 阅读关系详情（只读）
- 导航到相关 Cell（只读）

**禁止操作**：
- 创建关系
- 编辑关系
- 删除关系
- 拖拽节点（暗示可编辑）
- 任何暗示"关系正在执行"的显示

**严格禁止的元素**：
- 当前步骤标识
- 下一步高亮
- 执行路径动画
- 进度指示器
- 流程控制按钮
- 任何颜色编码暗示"正在执行"
- 箭头暗示流向（使用无向线或双向箭头）

**明确说明的事实**：
在拓扑视图顶部必须显示：
> "This is a frozen declarative topology. Relationships represent structural dependencies, not execution order or runtime flow. Nothing is running."

**去时态化要求**：
- 不使用"当前" "下一步" "正在"等词汇
- 不使用箭头暗示流向（使用无向线或双向箭头）
- 关系标签使用名词（依赖 / 参考）而非动词（执行 / 触发）

---

### 3.5 Methodology（方法论视角 - 只读）

**实体定义**：
Methodology 是冻结时采用的方法论视角。在 Run-Time 中，Methodology 仅用于展示冻结时的认知上下文，不用于任何操作。

**必须呈现的字段**（全部只读）：
- 方法论名称（只读文本）
- 方法论描述（只读文本）
- 方法论状态（冻结时的状态，只读）

**呈现位置**：
- 界面顶部或侧边栏（只读标识）
- 以标签或只读文本形式呈现
- **禁止下拉选择器**（暗示可切换）

**操作允许**：
- 查看方法论详情（只读）

**禁止操作**：
- 切换方法论（冻结时的方法论不可更改）
- 创建新方法论
- 任何暗示"方法论正在生效"的显示

**明确说明的事实**：
在方法论标识旁必须显示：
> "This is the methodology perspective that was active when this structure was frozen. It is read-only and cannot be changed."

---

### 3.6 Freeze Record（冻结记录）

**实体定义**：
Freeze Record 是冻结操作的不可变记录，用于审计和追溯。

**必须呈现的字段**（全部只读）：
- 冻结时间戳（精确到秒）
  - **时间戳标注要求**：必须明确标注为"Frozen at [timestamp]"
  - **禁止使用的表述**：不得使用"Last updated"、"Updated at"、"Current"、"Latest"、"Recent"或任何暗示活动时间的表述
  - **时间戳性质**：该时间戳是静态的，代表冻结操作发生的时刻，不代表任何变化或活动，不会自动更新
- 冻结人标识（谁执行了冻结操作）
- 冻结快照 ID（用于引用）
- 冻结前 Draft ID（如果有）
- 冻结原因或备注（如果有记录）

**呈现位置**：
- 在 Company Identity Layer (L1) 中可见
- 或在独立的 Provenance Layer (L5) 中详细展示

**操作允许**：
- 查看冻结记录详情（只读）
- 复制冻结快照 ID（用于引用）

**禁止操作**：
- 修改冻结记录
- 删除冻结记录
- 任何暗示"冻结可以撤销"的显示

**明确说明的事实**：
在冻结记录区域必须显示：
> "This freeze record is immutable. The structure cannot be unfrozen or modified. To make changes, create a new Draft in Design-Time."

## 4. Run-Time 导航模型

### 4.1 导航结构

**重要说明**：以下导航是**主要视图导航**，用于视图切换，不是操作流程。用户可以按任何顺序访问这些视图，没有推荐顺序。

**主要视图导航（始终可见）**：
1. Company Overview（L1 - Company Identity）
2. Structure View（L2 - Cells + Roles）
3. Topology View（L3 - Declarative Relations）
4. Freeze Record（L5 - Provenance）

**全局标识（右上角或顶部）**：
1. Frozen Status Badge（"FROZEN - READ ONLY"）
2. Freeze Timestamp（始终可见）

**二级导航（上下文进入）**：
- 从 Cell 列表点击进入 Cell 详情（只读）
- 从 Cell 详情查看附属 Role（只读）
- 从 Topology 点击节点进入 Cell 详情（只读）
- 从 Topology 点击边查看关系详情（只读）

### 4.2 导航语义

**导航原则**：
- 导航是**阅读路径**，不是工作流
- 导航是**上下文切换**，不是状态转换
- 导航是**视角切换**，不是操作触发

**禁止的导航语义**：
- "下一步"按钮
- "继续"按钮
- "完成"按钮
- 进度指示器
- 步骤计数器
- 任何暗示"流程"的导航

**允许的导航语义**：
- "查看详情"
- "返回列表"
- "查看相关"
- "查看冻结记录"

### 4.3 深度链接

**要求**：
- 支持 URL 深度链接到特定实体（如 `/frozen/COMP-123/cell/CELL-456`）
- 深度链接不改变冻结状态
- 深度链接不触发任何操作
- 深度链接仅用于定位和分享

**禁止**：
- 深度链接触发编辑
- 深度链接改变状态
- 深度链接暗示"跳转到编辑模式"

## 5. 人类认知路径（阅读路径）

**重要说明**：以下认知路径是**示例阅读路径**，展示人类可能如何理解冻结结构。这些路径不是推荐顺序，不是最佳路径，不是引导流程。用户可以按任何顺序阅读。实现时不应暗示"应该按此顺序阅读"或"这是操作流程"。

### PATH A: 理解这个公司是什么

**目标**：回答"这个冻结的公司结构是什么？"

**路径步骤**：
1. **查看 Company Identity**
   - 阅读 Company 名称和描述
   - 确认冻结状态和时间戳
   - 理解这是哪个虚拟公司

2. **查看结构概览**
   - 浏览 Cell 列表（数量、名称）
   - 了解责任单元的整体分布
   - 查看方法论上下文

3. **形成整体认知**
   - "这是一个关于 X 的虚拟公司结构"
   - "包含 Y 个责任单元"
   - "在 Z 时间被冻结"

**界面支持**：
- Company Identity Layer 始终可见
- 结构概览卡片或摘要
- 明确的冻结状态标识

**禁止**：
- 暗示"下一步应该做什么"
- 显示"完成度"或"进度"
- 任何评价性语言

---

### PATH B: 检查责任边界

**目标**：回答"每个 Cell 的责任边界在哪里？"

**路径步骤**：
1. **浏览 Cell 列表**
   - 查看所有 Cell 名称
   - 识别需要深入检查的 Cell

2. **深入 Cell 详情**
   - 阅读责任描述（What）
   - 阅读责任边界（What NOT）
   - 查看附属 Role 约束

3. **验证边界清晰性**
   - 确认每个 Cell 都有明确的 What 和 What NOT
   - 确认 Role 约束与责任边界一致
   - 识别可能的边界模糊点（仅事实陈述）

**界面支持**：
- Cell 列表视图（可筛选、可搜索，但不排序或推荐）
- Cell 详情视图（完整展示 What / What NOT / Roles）
- 边界完整性事实（如"3 个 Cell 无边界声明"）

**禁止**：
- 推荐"应该检查哪些 Cell"
- 评分"边界清晰度"
- 暗示"某些 Cell 更重要"

---

### PATH C: 审计结构依赖

**目标**：回答"Cell 之间的声明性关系是什么？"

**路径步骤**：
1. **阅读拓扑概览**
   - 阅读拓扑画布（只读）
   - 识别关系类型分布
   - 阅读关系列表

2. **深入关系详情**
   - 阅读每个关系的类型和描述
   - 理解依赖、参考、输入输出关系
   - 验证关系的合理性

3. **检查结构完整性**
   - 识别孤立 Cell（无任何关系）
   - 识别关系链（如果有）
   - 验证关系的一致性（仅事实陈述）

### PATH D: 验证冻结合法性

**目标**：回答"这个冻结是否合法？是否可以信任？"

**路径步骤**：

   - 检查是否有不一致的地方（仅事实陈述）
   - 验证所有引用是否有效

3. **追溯设计决策**
   - 查看冻结时的结构状态
   - 理解冻结的原因（如果有记录）
   - 确认冻结的不可逆性

**界面支持**：

## 9. 界面视觉与交互原则

### 9.1 视觉定位

**设计参考**：
- 冻结的法律实体档案（或历史性法律实体记录）（权威性 / 不可变性 / 冻结性）
- 冻结的建筑蓝图（结构性 / 精确性）
- 历史文档查看器（只读性 / 审计性）
- 博物馆展品标签（静态性 / 信息性）

**布局要求**：布局应明确传达"这是冻结档案"，而非"活跃注册表"或"实时登记"。视觉风格应类似历史文档或博物馆展品，而非实时数据库。

**避免的参考**：
- 现代 SaaS 仪表盘（避免暗示活动）
- 监控工具（避免暗示实时数据）
- 控制面板（避免暗示可操作）
- 工作流引擎（避免暗示执行）

### 9.2 色彩原则

**主色调**：
- 中性色为主（灰 / 深蓝 / 浅灰）
- 避免绿色（暗示运行/成功）
- 避免红色（暗示错误/警告）
- 避免黄色（暗示警告/注意）
- 使用深色或锁定图标表示"冻结"

**强调色**：
- 冻结标识：深灰或锁定图标
- 只读标识：中性蓝或灰色
- 重要信息：深色文本，无高亮

**状态色**：
- Frozen：深灰或锁定图标
- Read-Only：中性蓝边框或背景

### 9.3 文案原则

**动词选择**：
- **优先使用**：阅读 / 检查 / 审计（明确传达只读性质）
- **谨慎或替换**：查看 / 浏览（如果可能被误解为监控或实时浏览，应避免使用）
- **禁止使用**：执行 / 运行 / 触发 / 启动 / 完成

**名词选择**：
- 使用：冻结结构 / 声明 / 快照 / 记录
- 避免：系统 / 运行环境 / 流程 / 步骤

**说明文字特征**：
- 客观陈述事实
- 使用过去时或现在时（描述冻结状态）
- 不使用将来时（避免暗示"将要发生什么"）
- 不使用评价性词汇

**示例允许的文案**：
- "This structure was frozen on [date]"
- "This Cell is responsible for [description]"
- "This relationship represents a dependency"

**示例禁止的文案**：
- "System is running"
- "This will execute"
- "Next step: [action]"
- "Quality score: 85%"

### 9.4 交互原则

**交互速度**：
- 所有交互应该是即时的（无延迟暗示"系统在处理"）
- 无加载动画（避免暗示"系统在工作"）
- 无进度指示器（避免暗示"操作进行中"）

**交互类型**：
- 点击查看详情（只读）
- 展开/折叠（仅显示控制）
- 导航切换（上下文切换）
- 搜索/筛选（仅显示控制，不推荐）

**禁止的交互**：
- 拖拽（暗示可编辑）
- 双击编辑（暗示可修改）
- 右键菜单（暗示操作）
- 自动保存（暗示活动）

### 9.5 布局原则

**布局特征**：
- 静态布局（无动画）
- 固定位置（关键信息始终可见）
- 清晰的信息层级
- 充足的留白（避免拥挤感）

**禁止的布局**：
- 动态更新区域（暗示实时数据）
- 浮动通知（暗示活动）
- 自动刷新区域（暗示活动）
- 实时图表（暗示数据流）
