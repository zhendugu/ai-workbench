## Implementation Guidelines

### What Implementation MUST Do

1. **Display Structural Facts**
   - Company anchor information
   - Cell responsibility structures
   - Declarative topology
   - Task Force collaboration
   - Methodology perspectives
   - Human decision points
   - Evidence backtracking

2. **Provide Navigation**
   - Primary navigation (Company, Responsibility Structure, Decision Points)
   - Secondary navigation (Topology, Task Force, Methodology)
   - Embedded capability (Evidence backtracking)

3. **Maintain Read-Only Discipline**
   - All data access is read-only
   - No file writes
   - No execution triggers
   - No modifications

### What Implementation MUST NOT Do

1. **No Execution Capabilities**
   - No execution buttons
   - No running indicators
   - No triggering operations

2. **No Recommendations**
   - No intelligent suggestions
   - No sorting by preference
   - No highlighting
   - No defaults

3. **No State Indicators**
   - No progress bars
   - No health indicators
   - No color coding (red/yellow/green)
   - No status animations

4. **No Implicit Behaviors**
   - No auto-fill
   - No smart defaults
   - No predictive behaviors
   - No learning mechanisms
