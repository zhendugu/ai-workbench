## Change Protocol

### To Modify Phase Q

1. **Create New Phase**: Assign new Phase ID (e.g. Q5, S1, T1)
2. **Document Changes**: Explicitly state what is modified and why
3. **Preserve Evidence**: Maintain all Phase Q evidence as reference
4. **New Validation**: Execute new validation under new Phase ID
5. **No Retroactive Changes**: Do not modify Phase Q evidence or conclusions

### To Extend Phase Q

1. **Create New Phase**: Assign new Phase ID
2. **Reference Phase Q**: Explicitly reference Phase Q as baseline
3. **Document Extension**: State what is extended and why
4. **New Validation**: Execute new validation under new Phase ID
5. **No Modification**: Do not modify Phase Q components
