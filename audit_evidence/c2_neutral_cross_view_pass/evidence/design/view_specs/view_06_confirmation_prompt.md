# View 06: Confirmation Prompt

**Document Status**: **DESIGN EVIDENCE**  
**Document Type**: View Specification (Neutral)  
**Effective Date**: 2025-12-26  
**Purpose**: Specifies a neutral confirmation prompt view

---

## View Purpose

Displays a confirmation prompt requiring explicit human selection. This view ensures human decision sovereignty.

---

## Display Rules

### Prompt Content

**Required Elements:**
- ✅ Clear statement: "Please select a pattern"
- ✅ Boundary reminder: "Presentation ≠ Recommendation"
- ✅ Explicit human selection required
- ✅ No default selection

**Forbidden:**
- ❌ Default selection
- ❌ Pre-selected option
- ❌ "Recommended" label
- ❌ "Suggested" label

### Option Display

**All Options:**
- ✅ Equal visual weight
- ✅ Same styling
- ✅ Same spacing
- ✅ Complete list (not truncated)

**Forbidden:**
- ❌ Highlighting certain options
- ❌ Emphasis on certain options
- ❌ Default selection
- ❌ Hidden options

---

## Interaction Rules

### User Actions

**Allowed:**
- ✅ Explicit selection of pattern (required)
- ✅ Cancel/back (explicit user action)
- ✅ Navigate to other views (explicit user action)

**Forbidden:**
- ❌ Auto-selection
- ❌ Default selection
- ❌ Pre-selection
- ❌ Auto-confirmation

---

## Neutrality Guarantees

**This View:**
- ✅ Explicit human selection required
- ✅ States "presentation ≠ recommendation"
- ✅ No default selection
- ✅ Clear human choice required
- ✅ No recommendation signals
